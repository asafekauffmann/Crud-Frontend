Organização e estrutura

O Projeto foi estrututado e organizado por modulos e componentes dinâmicos, atribuindo assim suas necessidades de estrutura e arquitetura especificas para SPA com o framework Angular mantendo Conceitos e boas práticas de programação.

Modulos existentes:
App-Module/LoginComponent; Abertura-Module; Read.me-Module; Nivel-Module; Desenvolvedor-Module; Shared-Module;

Base
Master-page (Base da estrutura do modulo com templates dinânimcos); Pages (Base de estrutura no modulo referente ao main); Services (base de estrutura no modulo referente aos serviços utilizados nos componentes); App-Routing (Base de estrutura de rota interna do módulo); App-Module (base de estrutura do modulo); Model (Interface para Api e Services);

No style.scss estará disponível as configurações de variáveis globais, configurações de classe LightMode e DarkMode.

No Shared-Module foi organizado o modulo de estrutura e serviços gerais dos componentes dinâmicos para utilização no sistema mantendo assim o padrão no projeto, Reaproveitamento de código, Clean code/architecture.

Página inicial do projeto - Login; Implementações template: Navegação lateral retraivel ao click em logo. Dark-Light Mode

Especificações Back-end

Back-end criada como uma API RESTful que utilize os métodos (​GET​, ​POST​, ​PUT/PATCH​ e ​DELETE​).

Dependências:
NodeJs - v.18 Prisma - v.4.11 TsNode - v.10.9 cors - v.2.8 Express - v.4.18 joi - v.17.8

Camadas
1 Rota - Define o nome do endpoint e qual controller sera chamado.
2 Controller - Valida os dados vindos da requisição, chama o serviço e fazer o retorno da requisição.
3 Service - Implementa regras de negocio e chamar o repositório responsavel por persistir os dados.
4 Repository - Fazer a implementação da tecnologia usada para se comunicar com o banco de dados.
